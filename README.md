# synca

Интерактивная синхронизация больших папок (архитектурные проекты, рендеры, модели) с внешним SSD для резервной копии. Один диалог-подтверждение, умное сравнение по содержимому, корзина для удалённых файлов.

## Установка (Windows)

1. Установите Node.js LTS с [nodejs.org](https://nodejs.org/) (выберите версию LTS, по кнопке).
2. (По желанию) Поставьте Windows Terminal из Microsoft Store — там лучше отображаются цвета.
3. Откройте PowerShell или Windows Terminal и перейдите в папку с synca:
   ```
   cd C:\путь\к\synca
   ```
4. Установите зависимости:
   ```
   npm install
   ```
5. Сделайте команду `synca` доступной из любой папки:
   ```
   npm install -g .
   ```
6. Закройте терминал и откройте заново (чтобы PATH обновился).

## Использование

В любом терминале наберите:

```
synca
```

**Первый запуск:** мастер спросит источник (откуда копировать) и назначение (куда складывать бэкап). Сохранит настройки и предложит синхронизировать.

**Каждый следующий запуск:**
1. Выбираете проект из списка
2. В главном меню: «Синхронизировать», «Найти дубликаты», «Настройки» или «Выход»
3. При синхронизации видите сводку (`+N новых`, `~N изменено`, `−N удалено`), можете посмотреть список и подтвердить
4. Прогресс-бар показывает процесс. Можно прервать `Ctrl+C` — прогресс сохранится, в следующий раз продолжит с того же места.

## Возможности

- **Атомарное копирование с проверкой:** каждый файл копируется во временный, fsync, переименование. После записи перечитывается и сверяется по хэшу xxhash — защита от тихих ошибок диска.
- **Умное сравнение:** сравнение идёт сначала по размеру и времени, потом для подозрительных файлов хэшируем содержимое — не перекопируем то, что не изменилось, даже если время скакнуло.
- **Корзина:** удалённые файлы не уничтожаются, а складываются в `.synca/trash/`. Лимит — 10% от размера бэкапа; когда лимит превышен, самое старое удаляется автоматически.
- **Поиск дубликатов:** в источнике ищет файлы с похожим именем и одинаковым содержимым, предлагает удалить лишние.
- **Логи всех операций:** последние 30 запусков лежат в `.synca/logs/` в виде JSON — что и когда копировалось.
- **Длинные пути на Windows:** автоматически используется префикс `\\?\` для путей длиннее 240 символов.
- **Заблокированные файлы:** если файл занят другой программой — три попытки с паузой, потом пропуск и запись в лог.

## Где что хранится

| Что | Где |
|---|---|
| Конфиг (список проектов) | `%APPDATA%\synca\config.json` |
| Манифест состояния | `<папка_назначения>\.synca\state.json` |
| Корзина (удалённые файлы) | `<папка_назначения>\.synca\trash\` |
| Логи операций | `<папка_назначения>\.synca\logs\` |

## Что делать если что-то пошло не так

- **Запустить мастер настройки заново:** удалите `%APPDATA%\synca\config.json`
- **Восстановить случайно удалённый файл:** загляните в `.synca\trash\` в папке бэкапа, файлы там лежат до 30 дней
- **Понять что произошло в последний раз:** откройте свежий лог в `.synca\logs\` — там видно какие файлы были скопированы, какие пропущены и почему

## Обновление

Скачайте новую версию (git pull или новый zip), затем в папке проекта:

```
npm install
npm install -g .
```

## Что НЕ делает

- Не следит за изменениями в фоне — каждая синхронизация запускается вручную
- Не двусторонняя синхронизация — источник всегда главный
- Не копирует символические ссылки
- Не загружает в облако
