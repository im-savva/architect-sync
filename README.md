# Архивариус (synca)

Полноэкранное консольное приложение для резервного копирования больших папок
(архитектурные проекты, рендеры, модели) на внешний SSD/флешку. Снэпшоты при
каждой синхронизации, откаты, восстановление файлов обратно на компьютер,
поиск дубликатов.

Команда в терминале — `synca`.

## Установка (Windows)

1. Установите Node.js LTS с [nodejs.org](https://nodejs.org/) (выберите версию LTS, по кнопке).
2. Поставьте Windows Terminal из Microsoft Store — в нём приложение выглядит как задумано.
3. Откройте Windows Terminal и перейдите в папку с программой:
   ```
   cd C:\путь\к\synca
   ```
4. Установите зависимости (сборка запустится автоматически):
   ```
   npm install
   ```
5. Сделайте команду `synca` доступной из любой папки:
   ```
   npm install -g .
   ```
6. Закройте терминал и откройте заново (чтобы PATH обновился).

## Использование

```
synca
```

**Первый запуск:** добавьте проект — название, рабочая папка на компьютере,
папка бэкапа на внешнем диске.

**Дальше:** выбираете проект и попадаете на главный экран с плитками:

- **Синхронизировать** — сравнение и копирование изменений на внешний диск.
  Перед применением — список изменений с фильтрами (`+` новые, `~` изменённые,
  `−` удаляемые), затем прогресс со скоростью и оценкой времени.
- **Вернуть на компьютер** — браузер бэкапа: можно вернуть файл, папку или всё
  целиком с внешнего диска в рабочую папку. У файла можно выбрать версию —
  текущую или из любого снэпшота (включая уже удалённые файлы).
- **История и откаты** — журнал всех запусков. Любую синхронизацию со
  снэпшотом можно откатить: внешний диск вернётся к состоянию «до». Откат
  тоже создаёт снэпшот, так что и его можно откатить.
- **Дубликаты** — поиск похожих по имени (`детская_1`, `детская_v2`,
  `детская_final`) или идентичных по содержимому файлов в рабочей папке.
  Все группы одним списком с галочками; найденное переезжает в корзину, не
  удаляется безвозвратно.
- **Настройки** — проекты, проверка после копирования.

Управление: стрелки — выбор, `Enter` — открыть/подтвердить, `Esc` — назад,
`Ctrl+C` — выход (можно в любой момент: прогресс сохраняется).

## Снэпшоты — главное

При каждой синхронизации старые версии **перезаписанных и удалённых** файлов
не пропадают, а уезжают в `.synca/snapshots/<дата-запуска>/` на внешнем диске.
Поэтому:

- случайно удалили файл на компьютере и синхронизировали — он в снэпшоте,
  возвращается через «Вернуть на компьютер»;
- перезаписали проект плохой версией — старая версия достаётся оттуда же;
- синхронизация целиком оказалась ошибкой — «История» → откат.

Лимит места: снэпшоты занимают не больше 10% от размера бэкапа (настраивается),
самые старые удаляются автоматически, но минимум 3 последних хранятся всегда.

## Возможности под капотом

- **Атомарное копирование с проверкой:** каждый файл копируется во временный,
  fsync, переименование. После записи перечитывается и сверяется по хэшу
  xxhash — защита от тихих ошибок диска.
- **Умное сравнение:** сначала по размеру и времени, потом для подозрительных
  файлов хэшируется содержимое — не перекопируем то, что не изменилось.
- **Прерывание без последствий:** `Ctrl+C` или `Esc` во время копирования —
  прогресс сохранён, следующий запуск продолжит с того же места.
- **Логи всех операций:** последние 60 запусков в `.synca/logs/` (JSON),
  они же — экран «История».
- **Длинные пути на Windows:** автоматический префикс `\\?\` для путей
  длиннее 240 символов.
- **Заблокированные файлы:** три попытки с паузой, потом пропуск и запись в лог.

## Где что хранится

| Что | Где |
|---|---|
| Конфиг (список проектов) | `%APPDATA%\synca\config.json` (Windows), `~/.config/synca/config.json` (Mac) |
| Манифест состояния | `<папка_бэкапа>\.synca\state.json` |
| Снэпшоты (старые версии) | `<папка_бэкапа>\.synca\snapshots\<дата>\` |
| Логи операций | `<папка_бэкапа>\.synca\logs\` |
| Корзина дубликатов | `<рабочая_папка>\.synca\trash\` |

## Если что-то пошло не так

- **Вернуть случайно удалённое/перезаписанное:** плитка «Вернуть на компьютер» —
  там видны и удалённые файлы, и старые версии по датам.
- **Понять, что произошло:** плитка «История» — каждый запуск со списком файлов.
- **Запустить мастер настройки заново:** удалите `config.json` (см. таблицу выше).

## Для разработки

```
npm run build          # пересобрать UI (esbuild, JSX → dist/)
npm run test:engine    # headless-прогон движка на песочнице
npm run dev            # сборка + запуск в dev-режиме
synca --dev            # dev-режим: отдельный конфиг + плитка «Тесты»
```

Песочница для ручного тестирования (`~/Downloads/synca-sandbox`, ~21 МБ
фейковых чертежей/рендеров с парами дубликатов):

```
node scripts/sandbox.mjs init     # создать заново
node scripts/sandbox.mjs mutate   # «рабочий день»: 2 изменены, 1 новый, 1 удалён
node scripts/sandbox.mjs status   # что где лежит
```

В dev-режиме то же самое доступно из приложения (плитка «Тесты»), конфиг
живёт отдельно (`config.dev.json`) и боевые настройки не трогает.

## Что НЕ делает

- Не следит за изменениями в фоне — каждая синхронизация запускается вручную
- Не двусторонняя синхронизация — рабочая папка на компьютере всегда главная
- Не копирует символические ссылки
- Не загружает в облако
